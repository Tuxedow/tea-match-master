# 奶茶品类爱好匹配网站 - 规格说明书

## 1. 项目概述

**项目名称**: 奶茶匹配大师
**项目类型**: 单页面Web应用
**核心功能**: 根据用户的身份、年龄、职业和爱好，智能匹配适合的奶茶品牌和单品
**目标用户**: 中国市场消费者，15-45岁人群

---

## 2. UI/UX 规格

### 2.1 布局结构

**页面分区**:
- **顶部导航栏** (60px高): Logo + 标题
- **Hero区域** (300px高): 欢迎语 + 简短介绍
- **表单填写区域** (卡片式): 用户信息输入
- **结果显示区域** (卡片式): 匹配结果展示
- **页脚** (50px高): 版权信息

**响应式断点**:
- 移动端: < 768px (单列布局)
- 桌面端: >= 768px (双列/网格布局)

### 2.2 视觉设计

**色彩方案**:
- 主色: #FF6B6B (珊瑚红 - 奶茶温暖感)
- 辅助色: #FFEAA7 (奶油黄 - 奶茶色调)
- 强调色: #D63031 (深红 - 按钮/高亮)
- 背景色: #FFF9F0 (暖白 - 奶茶白)
- 文字色: #2D3436 (深灰)
- 卡片背景: #FFFFFF
- 边框色: #E8D5C4

**字体**:
- 标题: "ZCOOL XiaoWei", "Ma Shan Zheng", serif (思源宋体备选)
- 正文: "Noto Sans SC", sans-serif
- 标题大小: 48px (H1), 28px (H2), 20px (H3)
- 正文大小: 16px
- 行高: 1.6

**间距系统**:
- 基础单位: 8px
- 卡片内边距: 24px
- 区域间距: 32px
- 元素间距: 16px

**视觉效果**:
- 卡片阴影: 0 4px 20px rgba(255, 107, 107, 0.15)
- 圆角: 16px (卡片), 8px (按钮/输入框)
- 悬停动画: transform: translateY(-4px), 0.3s ease

### 2.3 组件规格

**表单组件**:
- 下拉选择框: 圆角边框，悬停高亮
- 多选标签: 胶囊样式，可多选
- 提交按钮: 渐变背景(#FF6B6B → #D63031)，悬停放大

**结果显示组件**:
- 品牌卡片: 图标 + 品牌名 + 推荐理由
- 单品卡片: 奶茶图片占位 + 名称 + 口味标签 + 匹配度

**动效**:
- 页面加载: 淡入上滑 (fadeInUp)
- 结果展示: 依次弹出 (stagger 0.1s)
- 匹配度进度条: 动画填充

## 3. 功能规格

### 3.1 用户输入项

| 字段 | 类型 | 选项 | 默认 |
|------|------|------|------|
| 年龄段 | 单选 | 15岁以下 / 15-22岁 / 23-30岁 / 31-40岁 / 40岁以上 | - |
| 性别 | 单选 | 男 / 女 / 其他 | - |
| 职业 | 单选 | 学生 / 上班族 / 自由职业 / 家庭主妇/夫 / 退休 | - |
| 口味偏好 | 多选 | 甜系 / 奶系 / 茶系 / 果系 / 咖啡因 | 至少选1个 |
| 品牌偏好 | 多选 | 喜茶 / 奈雪的茶 / 霸王茶姬 / 古茗 / 书亦烧仙草 / 茶百道 / 益禾堂 / 阿水大杯茶 / CoCo都可 / 一点点 | 可不选 |
| 价格区间 | 单选 | 10元以下 / 10-20元 / 20-30元 / 30元以上 | - |
| 场景 | 多选 | 日常解渴 / 下午茶 / 拍照打卡 / 社交聚会 / 运动后 | 至少选1个 |
| 身体状况 | 多选 | 正常 / 乳糖不耐受 / 控糖 / 失眠体质 | 可不选 |

### 3.2 奶茶品牌数据库

```
品牌列表:
1. 喜茶 - 均价25-35元，定位高端，创新果茶
2. 奈雪的茶 - 均价25-35元，定位高端，软欧包+茶
3. 霸王茶姬 - 均价15-22元，定位中端，原叶鲜奶茶
4. 古茗 - 均价12-20元，定位中端，水果茶为主
5. 书亦烧仙草 - 均价10-18元，定位中低端，烧仙草特色
6. 茶百道 - 均价12-20元，定位中端，杨枝甘露热门
7. 益禾堂 - 均价8-15元，定位低端，学生党首选
8. 阿水大杯茶 - 均价10-18元，定位中低端，山东区域品牌
9. CoCo都可 - 均价12-22元，定位中端，经典款多
10. 一点点 - 均价12-20元，定位中端，隐藏菜单丰富
```

### 3.3 推荐单品数据库 (至少30款)

按品牌分类，包含:
- 产品名称
- 价格区间
- 口味标签 (甜度、茶底、配料)
- 适合人群标签
- 热量/无糖选项

### 3.4 匹配算法

**权重分配**:
- 口味偏好: 30%
- 价格区间: 25%
- 职业/场景: 20%
- 年龄段: 15%
- 身体状况: 10%

**匹配逻辑**:
1. 根据口味偏好筛选品牌
2. 根据价格区间筛选单品
3. 根据职业匹配场景需求
4. 根据年龄调整推荐风格
5. 根据身体状况过滤/标记

**输出**:
- 推荐品牌 (1-3个)
- 推荐单品 (每个品牌1-2款)
- 匹配度评分 (0-100%)
- 个性化推荐理由

### 3.5 边缘情况处理

- 未填写必填项: 提示完善信息
- 所有选项都不选: 使用默认推荐
- 无匹配结果: 返回相近选项
